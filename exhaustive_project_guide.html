Explorateur de Code : Arborescence Totale

Projet : 3D Human Motion Comparison System | Dernière MaJ : Février 2026

Racine & Configuration Globale

.env.example // Modèle pour les variables d'environnement (mots de passe, ports)
.gitignore // Empêche l'upload des dossiers lourds (venv, nodes_modules)
.gitattributes // Force le format des retours à la ligne (Windows/Mac)
README.md // Le manuel d'utilisation général
CLEAN_ARCHITECTURE_GUIDE.md // Manuel pour garder le code propre et organisé
TECH_STACK.md // Détail de toutes les technos utilisées
deploy.sh // Script pour automatiser le lancement du projet
docs/ // Dossier central de toute la doc technique

Frontend (Flutter Mobile/Web/Desktop)

Localisation : /frontend. Structure basée sur la Clean Architecture.

lib/
│ ├── main.dart // L'interrupteur qui allume l'application
│ ├── core/
│ │ ├── theme/ // Couleurs, Thème Dark/Light (app_theme.dart)
│ │ ├── network/ // Config des appels serveurs (Dio, API client)
│ │ ├── utils/ // Fonctions d'aide (formatage de date, calculs)
│ │ └── widgets/ // Boutons et champs de texte réutilisables
│ ├── features/
│ │ ├── starting/ // Splash screen et introduction au démarrage
│ │ ├── auth/ // Connexion (signin), Inscription (signup), Password perdu
│ │ ├── home/ // Le Dashboard (exercices, recherche, sidebar)
│ │ ├── analysis/ // Capture vidéo, Upload et Lecteur 3D
│ │ ├── history/ // Liste de toutes les anciennes analyses
│ │ └── settings/ // Profil utilisateur et réglages de l'app
│ └── l10n/ // Gestion des langues (Français, Anglais, Arabe)
assets/
│ ├── images/ // Logo, fonds d'écran (logo.png, network_bg.png)
│ └── icons/ // 12 icônes SVG (search, logout, user, trash, etc.)
pubspec.yaml // Le fichier maitre des dépendances Flutter

Backend (FastAPI Python)

Localisation : /backend. C'est ici que l'IA SMPL-X tourne.

app/
│ ├── main.py // Point d'entrée du serveur API
│ ├── api/ // Routes pour l'Auth, les Exercices et l'Analyse
│ ├── models/ // SQL Alchemy (Définit les tables de la base de données)
│ ├── db/ // Config de connexion et script SQL initial (db_config.py, schema.sql)
│ ├── schemas/ // Validation des données entrantes/sortantes (Pydantic)
│ └── services/ // Le "moteur" : traitement vidéo, conversion SMPL-X
scripts/
│ ├── init_database.py // Script pour créer les tables la première fois
│ └── load_training_data.py // Importe les exercices de référence dans la DB
smplx_utils/ // Adaptateur mathématique pour le modèle humain 3D
requirements.txt // Liste des bibliothèques Python nécessaires
venv/ // Dossier invisible contenant l'environnement Python

Dataset (Le Vault de Données)

Localisation : /dataset. Stockage physique des fichiers lourds.

videos/
│ ├── s03, s11, .../ // Dossiers par Sujet (S03 à S11)
│ │ ├── videos/ // Vidéos MP4 originales
│ │ ├── smplx/ // Paramètres de posture extraits
│ │ ├── joints3d_25/ // Coordonnées des articulations
│ │ └── camera_parameters/ // Données pour l'orientation 3D
│ └── user_uploads/ // Stocke les vidéos envoyées par les utilisateurs mobiles
smpl/
│ └── smplx/ // Les fichiers .pkl et .npz (MALE, FEMALE, NEUTRAL)
keypoints/ // Exports JSON rapides pour comparaison rapide

Dossiers Natifs (Générés par Flutter)

/android

Code spécifique Android (Manifest, icônes adaptables, permissions caméra).

/ios

Code spécifique Apple (Info.plist, clés de signature, configuration CocoaPods).

/windows

C++ pour faire tourner l'application en mode bureau (Desktop Windows).

/macos

Configuration pour le déploiement sur les ordinateurs Apple.
