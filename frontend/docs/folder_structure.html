🏗️ Structure du Projet

3D Human Motion Comparison System - Clean Architecture

🛠️ Stack Technologique

🔧 Backend

  • FastAPI (REST API)
  • SQLAlchemy (ORM)
  • PostgreSQL + JSONB
  • BlazePose (Pose Estimation)
  • PyTorch (SMPL-X)
  • Pydantic (Validation)

📱 Frontend

  • Flutter (Multi-platform)
  • flutter_bloc (State)
  • dio (HTTP Client)
  • flutter_gl (3D Rendering)
  • go_router (Navigation)
  • Clean Architecture

💾 Database & Storage

  • PostgreSQL 17
  • JSONB (Flexible Data)
  • Filesystem (Videos)
  • JSON (Keypoints/SMPL)
  • Docker (Deployment)

🔧 Backend (Python/FastAPI)

backend/app/api/
📡 Endpoints de l'API REST

Définit les routes HTTP que le frontend appelle pour communiquer avec le serveur.

  • auth.py - Routes d'authentification (signup, login, password reset)
  • analysis.py - Routes d'analyse de mouvement (upload, résultats)
  • exercises.py - Gestion des exercices de référence
@router.post("/api/auth/signup") async def signup(user_data: UserCreate): # Créer un nouvel utilisateur return {"message": "User created"}
backend/app/models/
🗄️ Modèles de base de données (SQLAlchemy)

Définit la structure des tables PostgreSQL avec les relations entre entités.

  • user.py - Table users (id, email, username, password_hash)
  • analysis.py - Table analyses (user_id, video_path, results)
  • exercise.py - Table exercises (name, category, reference_data)
class User(Base): __tablename__ = "users" id = Column(Integer, primary_key=True) email = Column(String, unique=True) analyses = relationship("Analysis")
backend/app/schemas/
✅ Schémas de validation (Pydantic)

Définit le format des données échangées entre frontend et backend (contrat d'API).

  • auth.py - UserCreate, UserLogin, Token, UserResponse
  • analysis.py - AnalysisCreate, AnalysisResponse
  • exercise.py - ExerciseResponse
class UserCreate(BaseModel): email: str username: str password: str
backend/app/services/
⚙️ Logique métier (Business Logic)

Contient la logique complexe de traitement (BlazePose, SMPL-X, comparaisons).

  • auth_service.py - Hash passwords, JWT tokens, emails
  • video_processor.py - BlazePose extraction, triangulation
  • smplx_service.py - SMPL-X fitting, mesh generation
backend/app/core/
🔐 Configuration & Sécurité

Paramètres globaux de l'application et fonctions de sécurité.

  • config.py - Variables d'environnement (DATABASE_URL, SECRET_KEY)
  • security.py - Password hashing, JWT creation/verification
backend/app/db/
🔌 Configuration base de données

Setup SQLAlchemy et connexion à PostgreSQL.

  • db_config.py - Engine, SessionLocal, Base
  • schema.sql - Schéma SQL (backup)

💾 Dataset (Stockage)

dataset/videos/
🎥 Stockage des vidéos MP4

Vidéos de référence (s03-s11) et vidéos uploadées par les utilisateurs.

  • reference/ - Vidéos d'entraînement (47 exercices × 8 sujets)
  • user_uploads/ - Vidéos uploadées par les utilisateurs
dataset/keypoints/
📍 Keypoints 3D (BlazePose)

Résultats de l'extraction de pose 3D (33 points par frame).

{ "frames": [ { "frame_id": 0, "keypoints": [ {"x": 0.5, "y": 0.3, "z": 1.2, "visibility": 0.95}, ... // 33 keypoints ] } ] }
dataset/smpl/
🎭 Paramètres SMPL-X

Modèles 3D et paramètres pour reconstruction du corps humain.

  • models/ - SMPLX_MALE.pkl, SMPLX_FEMALE.pkl, SMPLX_NEUTRAL.pkl
  • reference/ - Paramètres des exercices de référence
  • user_uploads/ - Paramètres des analyses utilisateurs
{ "betas": [0.1, -0.2, ...], // 10 shape params "body_pose": [[...], ...], // 63 pose params "global_orient": [[0, 0, 0]], // Rotation "transl": [[x, y, z], ...] // Translation }

🔄 Flux de Données - Upload Vidéo

Frontend (analysis/new_analysis_page.dart)
Utilisateur sélectionne vidéo → video_upload_widget.dart
Frontend (data/datasources/)
POST /api/analysis/upload via analysis_remote_datasource.dart
Backend (app/api/analysis.py)
Reçoit vidéo → Sauvegarde dans dataset/videos/user_uploads/
Backend (services/video_processor.py)
BlazePose extraction → Sauvegarde keypoints dans dataset/keypoints/
Backend (services/smplx_service.py)
SMPL-X fitting → Sauvegarde params dans dataset/smpl/
Backend (models/analysis.py)
Sauvegarde métadonnées dans PostgreSQL → Retourne analysis_id
Frontend (analysis/analysis_result_page.dart)
GET /api/analysis/{id} → model_3d_viewer.dart affiche le mesh 3D
