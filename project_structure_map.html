Où se trouve quoi ?

Guide complet de l'arborescence du projet Motion 3D

💡 Pourquoi ce guide ? Entre Flutter (Mobile), FastAPI (Serveur) et les modèles SMPL-X (IA), on peut vite se perdre. Voici une carte précise du "territoire" pour savoir exactement où coder quoi.

1. Le Frontend (Flutter)

C'est l'interface utilisateur. Multi-plateforme (iOS, Android, Web, Desktop). Tout se passe principalement dans lib/.

frontend/
├── android/, ios/, macos/,... // Configs par plateforme
├── assets/ // Images, Icônes et Polices
├── lib/
│ ├── main.dart // Le coeur qui lance tout
│ ├── core/ // Thème, API Client, constantes
│ └── features/ // Découpé par métier
│ ├── auth/ // Login / Inscription
│ ├── home/ // Dashboard principal
│ ├── analysis/ // Upload & Vue 3D (Le plus important)
│ └── settings/ // Profil & Options
└── pubspec.yaml // Liste des packages (Dio, Bloc, etc.)

Features

On utilise une archi propre (Clean Architecture). Chaque dossier a sa Data (API), son Domain (Règles) et sa Presentation (UI Widgets).

Core

Si tu veux changer la couleur principale de l'app ou l'adresse du serveur, c'est ici que ça se passe (app_theme.dart ou api_client.dart).

2. Le Backend (FastAPI Python)

Le cerveau qui fait les calculs lourds (IA) et gère la base de données.

backend/
├── app/
│ ├── main.py // Point d'entrée du serveur
│ ├── api/ // Les "Routes" (URL que Flutter appelle)
│ ├── models/ // Structure des tables SQL (SQLAlchemy)
│ ├── schemas/ // Format des données JSON (Pydantic)
│ └── services/ // Logique (Traitement vidéo, SMPL-X)
├── scripts/ // Setup de la DB & Import de données
├── smplx_utils/ // Adaptateurs pour le modèle 3D
└── requirements.txt // Libs Python (FastAPI, PyTorch...)

API / Routes

Chaque fichier correspond à un besoin : auth.py pour les comptes, analysis.py pour le traitement des vidéos.

Services

C'est ici qu'on appelle BlazePose ou qu'on génère le modèle SMPL-X à partir des vidéos reçues.

3. Le Stockage (Dataset & Docs)

Où dorment les vidéos et où on garde les preuves du design.

dataset/
├── videos/
│ ├── reference/ // Les vidéos "étalon"
│ └── user_uploads/ // Les vidéos envoyées par les gens
├── keypoints/ // Export JSON des articulations
└── smpl/ // Paramètres 3D finaux

docs/ // Diagrams UML et Database Design

Fichiers de Configuration (Racine)

  • 📄 .env : Cache tes secrets (mots de passe DB, clés API).
  • 📄 deploy.sh : Petit script pour lancer tout le système d'un coup.
  • 📄 README.md : Le mode d'emploi global.
